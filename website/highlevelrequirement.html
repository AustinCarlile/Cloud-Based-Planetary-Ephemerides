High-Level Requirements

Web Service Development:

  • Develop a Python-based RESTful web service to handle requests for generating ISD.
  • The service should interact with existing open-source software (like ALE) to process requests.
  • It should accept image labels and SPICE kernel quality parameters as input via web requests.

Data Optimization:

  • Design a format to reduce the size of the JSON-formatted ISD data, possibly through binary conversion, compression, or a combination of methods.
  • Ensure the new format can be converted back into a standard ALE-supported format.

Scaling and Performance:

  • Address the challenge of handling large-scale data processing, including thousands of simultaneous ISD requests.
  • Implement memoization to cache frequently queried data, reducing the time needed for data retrieval.
  • Develop a cache validation mechanism to update or invalidate cache entries when SPICE data is updated, ensuring data consistency.

Deployment on AWS:

  • Design the service to be compatible with AWS free-tier for easy deployment and testing.
  • Aim for a setup that can be quickly deployed using tools like Docker.

Stretch Goals:

  • Develop a queryable data store for ISD, allowing scientists to access and download cached ISD data more efficiently.
